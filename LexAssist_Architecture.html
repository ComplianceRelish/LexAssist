2. APIs Used for Legal Research & Analysis

APIProviderRole in Pipeline
Claude AI Anthropic Core analysis engine — 3-pass deep analysis, citation verification, chat, document drafting
Indian Kanoon indiankanoon.org Case law search (relevance + most-recent), full judgment text fetching, citation ground-truth
GPT-4o Vision OpenAI Document OCR — extracts text from scanned PDFs, images, handwritten documents
Supabase Supabase Auth, PostgreSQL database, row-level security, case diary storage
Local Statute DB Built-in Authoritative section text for 20 acts (IPC/BNS, CrPC/BNSS, CPC, Constitution, etc.)

3. Legal Research & Analysis Sequence

1
User Input — Brief text, file upload, or voice input submitted via frontend
↓
2
Regex Extraction — Extracts parties, dates, sections, articles, courts, FIR numbers from raw text
↓
3
Case Classification — Determines type (Criminal / Civil / Constitutional / Family / Commercial) and jurisdiction
↓
4
Indian Kanoon Search API
Pass A: Relevance-ranked results  |  Pass B: Most recent (last 3 years, sorted by date)
Full judgment text fetched for top 3 precedents (ratio decidendi)
↓
5
Statute Lookup Local DB — Matches extracted sections against authoritative statute text (20 acts, ~200 sections)
↓
6
Claude Pass 1 — Issue Identification API
Fast classification of legal issues and case type (skipped for briefs < 3000 chars)
↓
7
Claude Pass 2 — Deep Structured Analysis API
Full analysis with injected context: statute text + Indian Kanoon precedents (with judgment excerpts) + regex entities.
Produces: legal issues, precedents, arguments, strategy, limitation periods, court fees, old↔new code mapping
↓
8
Claude Pass 3 — Citation Verification API
Layer 1: Cross-reference AI citations against Indian Kanoon ground-truth
Layer 2: AI verification for citations not in Kanoon results — flags unverified with ⚠️
↓
9
Response Delivery — Structured JSON streamed via SSE to frontend → displayed in tabbed UI
